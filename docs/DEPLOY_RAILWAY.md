# Развертывание бота на Railway 🚂

Railway — это отличный сервис для хостинга ботов. Там можно запускать код 24/7 и управлять переменными окружения. Мы специально подготовили бота для легкого деплоя!

## Подготовка (уже сделана ✅)
Мы только что обновили проект для Railway:
1. Добавили поддержку переменной `GOOGLE_CREDENTIALS_JSON` (чтобы не заливать файл с ключами).
2. Заложили файл конфигурации `railway.json`, чтобы сервис знал, как запускать бота (`python bot.py`).

## Шаг 1. Создаем проект в Railway
1. Зарегистрируйтесь на [Railway.app](https://railway.app/) (можно через GitHub).
2. Нажмите **New Project** -> **Deploy from GitHub repo** (и выберите ваш репозиторий `finance-bot`).
   *Либо можно загрузить код с компьютера через утилиту `railway up`*.

## Шаг 2. Заполняем секретные ключи
Бот не заработает без ключей. В панели вашего проекта (в Railway) перейдите на вкладку **Variables** и добавьте туда переменные из вашего `.env`:

* `TELEGRAM_BOT_TOKEN`
* `OPENROUTER_API_KEY`
* `AI_MODEL` (например, `google/gemini-2.5-flash`)
* `SPREADSHEET_NAME`
* `CURRENCY`
* `ALLOWED_USERS`

### И самое главное — ключи Google:
Вместо файла используйте новую переменную:
1. Создайте переменную `GOOGLE_CREDENTIALS_JSON`.
2. Откройте ваш локальный файл `credentials.json` с компьютера с помощью текстового редактора.
3. Скопируйте **весь текст** из него (от `{` до `}`) и вставьте как значение (Value) этой переменной. 

## Шаг 3. Запуск 🚀
Как только вы добавите переменные, Railway автоматически перезапустит билд. 
Через пару минут бот будет запущен и начнет работать 24/7! За логами можно следить во вкладке **Deployments** -> **View Logs**.
