# Подробное руководство по настройке (Onboarding) 🚀

Добро пожаловать! Если вы здесь, значит, вы хотите запустить своего личного финансового ИИ-ассистента в Telegram. 
Этот документ — максимально подробная, пошаговая инструкция для "чайников". Мы разберем всё: от создания бота до настройки Google Cloud и нейросетей.

---

## 🗺️ Как работает бот? (Кратко)

1. Вы пишете сообщение боту в **Telegram** (например: *"Потратил 500 рублей на такси и 1500 продуктов"*).
2. Бот отправляет ваш текст в **OpenRouter** (агрегатор нейросетей, где работает умный AI).
3. Нейросеть понимает, что это за траты, распределяет по категориям и возвращает структурированные данные.
4. Бот записывает эти данные в вашу личную **Google Таблицу** (Google Sheets).

Для этого нам нужно связать все три сервиса воедино.

---

## Шаг 1: Создание Telegram бота и получение ID

Нам нужно создать самого бота и узнать ваш личный ID, чтобы чужие люди не могли записывать траты в вашу таблицу.

### 1.1 Получаем токен бота
1. Откройте Telegram и в поиске найдите **@BotFather** (с синей галочкой).
2. Нажмите `Запустить` (или напишите `/start`).
3. Отправьте команду `/newbot`.
4. BotFather попросит придумать **имя** (например, `Мой Финансист`).
5. Затем попросит придумать **юзернейм** — он должен заканчиваться на `_bot` или `Bot` (например, `Ivan_FinanceBot`).
6. В ответ вы получите длинное сообщение. Найдите там строку, которая выглядит так: `1234567890:AAH_XXXXXXXXXXXXXXX` — это ваш **TELEGRAM_BOT_TOKEN**. Скопируйте его и сохраните где-нибудь в блокноте.

### 1.2 Узнаем свой ID
1. В поиске Telegram найдите бота **@userinfobot** или **@GetIDs Bot**.
2. Нажмите `Запустить`.
3. Бот пришлет вам сообщение с вашим ID (число, например `123456789`).
4. Скопируйте его — это будет ваш **ALLOWED_USERS** (чтобы только вы могли пользоваться ботом).

## Шаг 2: Настройка Google Cloud и Google Таблиц 🔥

*Это самый сложный, но самый важный шаг. Будьте внимательны.*
Бот не может просто так менять любые ваши таблицы. Ему нужен специальный невидимый аккаунт-робот (Service Account), которому вы дадите доступ к своей таблице.

### 2.1 Создаем проект и включаем API
1. Перейдите в [Google Cloud Console](https://console.cloud.google.com/).
2. Войдите в свой Google аккаунт. Перейдите в раздел консоли.
3. В левом верхнем углу (рядом с логотипом Google Cloud) нажмите на **"Select a project"** (Выбрать проект) -> Нажмите **"New Project"** (Новый проект).
4. Назовите его, например, `FinanceBot`. Нажмите **Create** (Создать).
5. Подождите пару секунд, пока проект создастся, и выберите его.
6. В боковом меню слева найдите **APIs & Services** (API и сервисы) -> **Library** (Библиотека).
7. В строке поиска введите `Google Sheets API`. Выберите его и нажмите **Enable** (Включить).
8. Снова вернитесь в **Library**, введите `Google Drive API` и тоже нажмите **Enable**. *(Без Drive API бот не сможет находить и создавать таблицы).*

### 2.2 Создаем аккаунт-робота (Service Account)
1. В боковом меню перейдите в **APIs & Services** -> **Credentials** (Учетные данные).
2. Нажмите синюю кнопку **+ CREATE CREDENTIALS** (Создать учетные данные) сверху.
3. Выберите **Service account** (Сервисный аккаунт).
4. Задайте имя, например `bot-sheets`. Нажмите **Create and Continue** (Создать и продолжить).
5. Шаги 2 и 3 можно пропустить (оставить пустыми). Просто нажмите **Done** (Готово).

### 2.3 Получаем файл-ключ `credentials.json`
1. Вы вернетесь на страницу Credentials. Внизу, в разделе **Service Accounts**, вы увидите только что созданный аккаунт. Нажмите на его Email (он выглядит как `bot-sheets@financebot-123.iam.gserviceaccount.com`).
2. Вверху перейдите на вкладку **KEYS** (Ключи).
3. Нажмите **Add Key** (Добавить ключ) -> **Create new key** (Создать новый ключ).
4. Выберите формат **JSON** и нажмите Create.
5. Файл автоматически скачается на ваш компьютер! **Переименуйте этот файл строго в `credentials.json`** и положите его в папку с проектом (туда, где лежит `bot.py`).

### 2.4 ВАЖНО: Даем боту доступ к Таблице
У вашего сервисного аккаунта есть e-mail (вы его видели в п. 2.3).
1. Откройте Google Таблицы под своим обычным аккаунтом.
2. Создайте новую таблицу (или откройте ту, куда хотите записывать траты). Запомните её название (например, `Финансовый учёт`).
3. Нажмите зеленую кнопку **"Настройки доступа"** (Share) в правом верхнем углу.
4. Вставьте e-mail вашего сервисного аккаунта (например, `bot-sheets@financebot-12345.iam.gserviceaccount.com`) и выдайте ему права **"Редактор"** (Editor).
5. Нажмите Отправить/Выдать доступ. Всё! Теперь бот может в нее писать.

## Шаг 3: Настройка нейросети (OpenRouter) 🧠

Мы используем платформу OpenRouter, так как она дает доступ к десяткам нейросетей (GPT-4, Claude, Gemini и др.) через один API ключ. Причем многие модели абсолютно бесплатны.

1. Зарегистрируйтесь на [OpenRouter.ai](https://openrouter.ai/).
2. Перейдите в раздел **Keys** ([открыть ссылку](https://openrouter.ai/keys)).
3. Нажмите **Create Key**. Введите любое название (например `FinanceBot`) и создайте.
4. Появится длинный ключ: `sk-or-v1-xxxxxxxxxxxx...`. **Скопируйте его прямо сейчас**, он показывается только один раз! Это ваш **OPENROUTER_API_KEY**.

> **Какую модель выбрать?**
> В настройках бота по умолчанию указана `openai/gpt-4o-mini` (она быстрая, дешевая и очень умная).
> Если вы хотите полностью **бесплатную** модель без ввода карты, используйте: `google/gemini-2.5-flash` или `meta-llama/llama-3-8b-instruct:free`.

## Шаг 4: Разворачиваем проект (Настройка и запуск) 💻

Теперь собираем всё вместе на компьютере или сервере. У вас должен быть установлен Python (версии от 3.10 и выше).

### 4.1. Подготовка папки
1. Откройте терминал (или командную строку).
2. Перейдите в папку с проектом: `cd путь/до/папки/finance-bot`
3. Создайте виртуальное окружение: `python -m venv venv`
4. Активируйте его:
   - Mac/Linux: `source venv/bin/activate`
   - Windows: `venv\Scripts\activate`
5. Установите зависимости: `pip install -r requirements.txt`

### 4.2. Настройка файла переменных окружения `.env`
1. В папке проекта создайте файл с названием `.env` (просто точка энв) или скопируйте из шаблона `cp .env.example .env`.
2. Откройте `.env` в любом текстовом редакторе.
3. Заполните его теми данными, которые мы собирали на предыдущих шагах:

```env
# Телеграм токен из Шага 1
TELEGRAM_BOT_TOKEN=1234567890:AAH_xxxxxxxxxxxxx

# Ключ от нейросети из Шага 3
OPENROUTER_API_KEY=sk-or-v1-xxxxxxxxxxxxxxxxx

# Ваша модель (рекомендуемая бесплатная)
AI_MODEL=google/gemini-2.5-flash 

# Имя файла с ключами (который мы скачали в Шаге 2)
GOOGLE_CREDENTIALS_FILE=credentials.json

# Название вашей таблицы в Google Sheets (куда вы выдали доступ)
SPREADSHEET_NAME=Финансовый учёт

# Валюта по умолчанию (например: RUB, USD, EUR, UZS)
CURRENCY=RUB

# Ваш Telegram ID из Шага 1 (чтобы бот игнорировал чужих людей)
ALLOWED_USERS=123456789
```

*Не забудьте убедиться, что файл `credentials.json` лежит в одной папке с `bot.py` и `.env`!*

### 4.3. Запуск!
В терминале пишем:
```bash
python bot.py
```
Если всё сделано правильно, в терминале не появится красных ошибок, а бот в Телеграме начнет отвечать на ваши команды (отправьте ему `/start` или просто напишите "купил кофе за 200").

Если вы выключите терминал, бот уснет. Чтобы он работал 24/7, переходите к Шагу 5.

## Шаг 5: Как запустить бота 24/7 бесплатно (Деплой на Railway) ☁️

**Railway** — это сервис, который позволяет хостить приложения и ботов в облаке. Код уже специально подготовлен для того, чтобы развернуть бота в пару кликов без лишних заморочек!

1. Зарегистрируйтесь на **[Railway.app](https://railway.app)** (можно через GitHub аккаунт).
2. Нажмите **New Project** -> **Deploy from GitHub repo** и выберите ваш скопированный репозиторий бота (убедитесь, что вы сделали форк репозитория).
3. Railway скачает код и попытается его собрать.
4. **Важно:** Бот упадет при первом запуске, так как не найдет ваших ключей. 
5. В Railway откройте карточку вашего запущенного сервиса (`findo-bot`), перейдите во вкладку **Variables**.
6. Создайте все переменные из вашего локального файла `.env` (Токен, API ключ, название таблицы, модель).
7. **Как быть с `credentials.json`?** 
   В облако нельзя загружать файлы с паролями напрямую. Создайте там специальную переменную **`GOOGLE_CREDENTIALS_JSON`**. Откройте ваш файл `credentials.json` на компьютере как текст, скопируйте **ВЕСЬ** его текст (включая скобки `{ }`) и вставьте в значение этой переменной. 
8. Как только вы сохраните переменные, Railway автоматически перезапустит билд, и бот начнет круглосуточную работу!

## Что делать, если что-то не работает? (Частые ошибки)

1. **Бот игнорирует сообщения:**
   Проверьте, правильно ли вы указали свой ID в `ALLOWED_USERS` в файле `.env`. Если вы запустили бота до добавления ID, перезапустите его (`Ctrl+C` в терминале и снова `python bot.py`).

2. **Ошибка `SpreadsheetNotFound` или бот не может найти таблицу:**
   Вы забыли выдать доступ сервисному e-mail (см. **Шаг 2.4**). Убедитесь, что название таблицы в `.env` (`SPREADSHEET_NAME`) совпадает буква в букву с названием в Google Sheets.

3. **Ошибка `File credentials.json not found`:**
   Вы не положили файл с ключами из Google Cloud в папку проекта, либо назвали его по-другому (например, `project-keys-123.json`). Переименуйте строго в `credentials.json`.

4. **Бот пишет об ошибке OpenRouter / API:**
   Закончились деньги/лимиты на балансе (если используете платную модель) или неправильно скопирован `OPENROUTER_API_KEY`. Замените `AI_MODEL` на какую-нибудь бесплатную, например `google/gemini-2.5-flash`.
